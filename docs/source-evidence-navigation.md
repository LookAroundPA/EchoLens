# 来源证据与时间点导航

EchoLens 会把搜索结果和关键观点尽量关联到现有 Faster-Whisper 时间戳分段，让阅读结果可以回到原始音频位置。

## 视频详情深链

视频详情支持两个可选查询参数：

```text
/videos/{id}?t=65.2&segment=12
```

- `t`：音频秒数；
- `segment`：转写片段索引。

页面会定位播放器、滚动到对应转写片段并尝试播放。浏览器阻止自动播放时，页面会保留定位并提示手工点击播放器。

播放器下方的“复制时间链接”会生成当前播放位置的可分享地址。

## 关键观点来源

`GET /api/videos/{id}` 的 `keyPointEvidence` 包含自动匹配结果：

```json
{
  "keyPointIndex": 0,
  "segmentIndex": 12,
  "segmentCount": 2,
  "start": 65.2,
  "end": 78.4,
  "text": "对应的原始转写片段",
  "score": 0.83
}
```

匹配只使用已经保存的 `segments_json`，不调用模型，也不修改数据库。每条观点会比较一至三个连续片段，低于最低匹配分数时不返回来源，避免强行关联。

## 搜索命中来源

`GET /api/search` 的每个结果包含 `match`：

```json
{
  "matchType": "transcript",
  "text": "包含搜索词的转写片段",
  "start": 65.2,
  "end": 78.4,
  "segmentIndex": 12,
  "segmentCount": 2
}
```

当关键词命中转写时，搜索页面可直接打开对应时间点。若只命中描述、摘要、标签或关键观点，则仍会返回命中文本，但不会伪造时间戳。

## 导出

JSON 导出会完整保留 `keyPointEvidence`。Markdown 导出会在可匹配的关键观点后附加来源时间范围：

```text
1. 人工智能可以减少重复劳动（来源 1:05–1:13）
```

## 当前边界

该功能是确定性文本匹配，不是语义向量检索。措辞差异很大的观点可能没有来源，后续语义检索完成后可替换或补充当前匹配逻辑。
