# 创作者知识档案

创作者详情页会基于该创作者已经完成转写和分析的视频，实时生成一个精简知识档案。

档案严格使用 `creator.sec_uid` 聚合，不依赖昵称，因此创作者改名不会拆成多个档案，同名创作者也不会被合并。

## 包含内容

`GET /api/creators/{sec_uid}` 在原有 `creator`、`topTags` 和 `videos` 字段之外，增加 `profile`：

```json
{
  "profile": {
    "overview": "创作者当前已沉淀 12 条完成内容……",
    "analyzedVideoCount": 12,
    "mainThemes": [
      {"tag": "AI", "count": 8}
    ],
    "insights": [],
    "representativeVideos": [],
    "recentVideos": []
  }
}
```

### 内容概览

概览由已完成视频数量、主要主题和代表观点组合生成，不额外调用 DeepSeek，也不会产生新的模型费用。

### 主要主题

主题来自现有分析标签，按出现频率排序。前端点击主题后，会进入限定该创作者和主题的搜索结果。

### 代表观点

系统会对各视频的关键观点做轻量文本归一化和近似去重：

- 相同或高度相似的表述会合并；
- `occurrenceCount` 表示该观点出现于多少个不同视频；
- 每条观点最多展示四个视频来源；
- 若能匹配到转写时间戳，来源包含 `start`、`end` 和 `segmentIndex`；
- 点击来源可以直接打开视频并播放对应片段。

该合并是确定性文本匹配，不是语义向量模型。措辞差异很大但含义相同的观点，第一版可能仍会分开显示。

### 代表视频

代表视频优先选择以下信息较完整的内容：

- 有摘要；
- 标签较丰富；
- 关键观点较多；
- 观点能够回溯到转写时间点。

每个代表视频会返回 `reason`，说明被选中的主要依据。

### 最近内容与全部时间线

- `recentVideos`：最近的已完成内容；
- `videos`：该创作者当前返回范围内的全部视频，包括处理中和失败状态。

## 数据更新

创作者档案不单独持久化。以下操作完成后刷新页面即可看到变化：

- 新视频完成分析；
- 人工修改摘要、标签或关键观点；
- 重新分析视频；
- 视频处理状态发生变化。

当前默认最多读取 100 条创作者视频用于页面和档案聚合，可通过接口的 `limit` 参数调整，最大 500。
